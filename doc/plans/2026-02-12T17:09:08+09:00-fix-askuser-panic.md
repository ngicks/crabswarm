# Fix askUserModel panic on question completion

## Context

After implementing the audit log panel, the TUI crashes with `index out of range [2] with length 2` when responding to an `AskUserQuestion` prompt. The panic occurs in `askUserModel.currentQuestion()` (`askuser.go:44`) called from `askUserModel.View()` (`askuser.go:195`).

**Root cause:** `advanceQuestion()` (`askuser.go:163-164`) increments `m.currentQ` past the end of `m.input.Questions`. Before the returned `promptCompleteMsg` command is processed by bubbletea, `View()` is called, which calls `currentQuestion()` with the out-of-bounds index. Classic bubbletea race: model state transitions to "done" but `View()` doesn't account for it.

Pre-existing bug unique to `askUserModel` — the only model iterating a dynamic question list. `permissionModel` and `exitPlanModel` use static choice slices and are safe.

## Fix

**File:** `hook/internal/tui/askuser.go`

1. Add `completed bool` field to `askUserModel` struct (line 13)
2. In `advanceQuestion()` (line 163): set `m.completed = true` when all questions answered
3. In `Update()` (line 52): early return if `m.completed` (prevents key events reaching `currentQuestion()` after completion)
4. In `View()` (line 184): early return with "Completing..." if `m.completed`

```go
// struct addition
type askUserModel struct {
    // ... existing fields ...
    completed bool
}

// advanceQuestion - add m.completed = true
func (m askUserModel) advanceQuestion() (askUserModel, tea.Cmd) {
    m.currentQ++
    m.cursor = 0
    m.selected = make(map[int]bool)
    if m.currentQ >= len(m.input.Questions) {
        m.completed = true
        // ... rest unchanged
    }
    return m, nil
}

// Update - add guard
func (m askUserModel) Update(msg tea.Msg) (askUserModel, tea.Cmd) {
    if m.completed {
        return m, nil
    }
    // ... rest unchanged
}

// View - add guard
func (m askUserModel) View() string {
    if m.completed {
        return headerStyle.Render("AskUserQuestion") + "\n\n" +
            progressStyle.Render("  Completing...") + "\n"
    }
    // ... rest unchanged
}
```
